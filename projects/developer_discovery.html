배포 과정 & 인프라 설정

1. Backend 깃허브 레포에서 Github Actions 를 통해 백엔드 Docker 이미지를 ECR에 푸쉬한다.

2. 이미지 빌드가 다 되면 Github Actions 중 Ec2 Deploy actions를 통해서 Ec2 배포 과정이 진행된다.

- CodeDeploy가 실행되고, before_install 스크립트를 통해서 EC2에 필요한 라이브러리 등을 다운 받는다.

- 이후 after_install 스크립트로 기존 docker 컨테이너를 내리고, ECR의 최신 이미지를 pull -> Docker run 하는 방식의 스크립트가 실행된다.

프로젝트를 진행하면서..

Django 에서 JWT 구현 방식과 기능을 위한 다양한 라이브러리를 접할 수 있어서 좋았습니다. 저 혼자 프론트, 백엔드, 데브옵스를 각각 진행하게 되면서 각 직무에 대한 삽질, 경험을 늘리게 된 것 같습니다. 다만, 기획과 디자인을 따로 지정하지 않아서 목적이 추상적이었고, 마땅한 문서도 작성하지 않아 진행이 조금 느렸던 것 같습니다. 내심 기획과 디자이너의 중요성을 다시 느끼게 되었습니다.

백엔드에서는 전체적으로 API 구성, 테스트 조건을 잡는 작업이 힘들었습니다. TDD 방식이 아니고 그렇다고 BDD 방식도 아니라, 즉흥적으로 개발하다보니 다시 보충해야하는 이슈가 있었습니다. 각 테스크에 대해서 정확한 목표 설정과 일정관리를 하지 못한 점이 아쉽습니다.

프론트에서는 JWT를 어떻게 저장할지와 Hydrate 이슈가 있어 고생한 경험이 있습니다. 덕분에 JWT 에 대해서 백엔드 입장 뿐만아니라 프론트에서 보통 어떤 방식을 이용하는지 확인 할 수 있는 좋은 경험이있습니다.

데브옵스에서는 Terraform을 이용해서 코드를 이용한 관리를 구성했는데요. 덕분에 이전 회사에서 배운 것들을 제 기술로 완전히 터득하게 되었습니다. 조금 아쉬운 것은 개인 프로젝트라서 다른 데브옵스 팀원과 어떤식으로 코드를 관리하는지 궁금하다는 점이 있습니다.
